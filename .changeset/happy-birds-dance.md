Add vanilla JS widget support and preview configuration helpers to Storybook tooling.

- **What** New `createPreviewConfig(options)` helper provides a simplified Storybook `preview.mjs` setup with automatic widget name/version validation via `@esmj/schema`. New `createVanillaRenderer()` (no arguments) creates a `render`/`update` pair for vanilla JavaScript widgets; stories supply the view via `args.component` as a function returning an HTML string, and event listeners are re-bound by calling `args.component.bindEventListeners(widget, container)` after each render. A per-widget `WeakMap` stores the container and view function, keeping state isolated across concurrent stories without touching the sealed widget object. `createWidgetLoader` reuses the existing widget instance when the story args are deeply equal (snapshot comparison), and unmounts and remounts only when args differ or the story changes. `state` and `props` are now applied after `widget.mount()` so the `load()` lifecycle cannot silently discard story-provided values. All exported functions are documented with JSDoc.
- **Why** Previously only Preact-based widgets had first-class Storybook support. Vanilla JS widgets had no renderer, forcing teams to wire up lifecycle management by hand. The missing `createPreviewConfig` abstraction caused boilerplate duplication across projects, and the lifecycle bugs in `createWidgetLoader` led to stale widget instances between story navigations.
- **How** Import the helpers from `@merkur/tool-storybook`. Call `createVanillaRenderer()` with no arguments in `.storybook/preview.mjs`; pass `component: MyViewFn` in story `args` and attach `MyViewFn.bindEventListeners = bindEventListeners` on the function for event re-binding (the idiomatic place is `View.js` itself). Use `createPreviewConfig({ widgetProperties, render })` to register the widget and obtain the loader. **⚠ Requires Storybook ≥ 10**: this package imports `storybook/preview-api` and `storybook/internal/core-events` at runtime; installing alongside Storybook < 10 will cause a module-not-found error at load time.
